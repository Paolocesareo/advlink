# Advlink ads.txt generator

Tool CLI standalone Node.js per generare file `ads.txt` IAB-compliant (spec 1.0.2) per publisher editoriali.

**Non** e integrato nel sito Next.js di Advlink — vive in `scripts/ads-txt/` ed e eseguibile direttamente con `node`. Nessuna dipendenza npm esterna, solo Node standard library.

Scopo immediato: generare i file `ads.txt` per le ~30 testate OPS Idea problematiche prima dell'onboarding Advlink.

## Prerequisiti

- Node.js 20 o superiore (testato su Node 22)
- Nessun `npm install` richiesto

## Uso single-domain

```bash
node scripts/ads-txt/generate.js gazzettadimilano.it
# Output: scripts/ads-txt/output/gazzettadimilano.it.txt
```

Con config custom e output path custom:

```bash
node scripts/ads-txt/generate.js gazzettadimilano.it \
  --config scripts/ads-txt/config-cliente-opsidea.json \
  --out /tmp/ads-gazzettadimilano.txt
```

## Uso batch

File CSV con 1 colonna `domain` (header + un dominio per riga):

```bash
node scripts/ads-txt/generate.js --batch scripts/ads-txt/publishers.example.csv
# Output: un .txt per ogni domain in scripts/ads-txt/output/
```

Con directory output custom:

```bash
node scripts/ads-txt/generate.js \
  --batch ops-idea-publishers.csv \
  --config config-cliente-opsidea.json \
  --out ./dist-adstxt/
```

## Configurare un config custom

1. Copia `scripts/ads-txt/config.default.json` → `config-cliente-opsidea.json` (nome a piacere, meglio fuori dal repo se contiene account_id reali sensibili).
2. Sostituisci ogni `"<<REPLACE_ME>>"` con l'`account_id` fornito dal publisher per quello SSP.
3. Rimuovi le entry non applicabili o aggiungine di nuove (schema: `domain`, `account_id`, `relationship`, `tag_id?`).
4. Esegui il generator con `--config <path>`.

Schema di una entry:

```json
{
  "domain": "google.com",
  "account_id": "pub-1234567890123456",
  "relationship": "DIRECT",
  "tag_id": "f08c47fec0942fa0",
  "comment": "Opzionale, solo per leggibilita JSON, NON finisce nel file ads.txt"
}
```

- `domain` (string, obbligatorio): dominio dello SSP (es. `google.com`).
- `account_id` (string, obbligatorio): id del publisher sullo SSP. Se vale `<<REPLACE_ME>>` o e vuoto la riga viene skippata con warning su stderr.
- `relationship` (string, obbligatorio): `DIRECT` oppure `RESELLER`.
- `tag_id` (string, opzionale): Certification Authority ID (16 char hex). Se omesso la riga e a 3 campi (valida IAB).

## Workflow OPS Idea (30 testate)

1. Raccogli da Sciscione/publisher la lista degli SSP attivi e gli account_id reali per ciascuno.
2. Prepara `config-opsidea.json` popolando i placeholder.
3. Prepara `publishers-opsidea.csv` con tutti e 30 i domini:
   ```csv
   domain
   testata-uno.it
   testata-due.it
   ...
   ```
4. Esegui:
   ```bash
   node scripts/ads-txt/generate.js \
     --batch publishers-opsidea.csv \
     --config config-opsidea.json \
     --out ./dist-adstxt-opsidea/
   ```
5. Consegna i 30 file a Giovanni Sciscione (o deploy diretto sui domini, ciascuno in `https://<domain>/ads.txt`).

## Verifica output

Il file `ads.txt` va caricato alla root del dominio publisher:
`https://<domain>/ads.txt`

Strumenti di validazione IAB:
- https://adstxt.guru
- https://ads-txt-manager.iabtechlab.com

## Comportamento del tool

- **Dominio invalido** → errore su stderr, exit 1, file non scritto.
- **Entry con `account_id` placeholder o vuoto** → warning su stderr, entry skippata, il file viene comunque scritto con le entry valide (puo anche avere 0 righe dati).
- **Config mancante o JSON invalido** → errore su stderr, exit 1.
- **Batch con qualche dominio invalido** → i domini validi vengono generati, quelli invalidi segnalati su stderr, exit finale 1.

## Formato output

File `ads.txt` tipico generato:

```
# ads.txt for gazzettadimilano.it
# Generated by Advlink ads-txt generator
# Date: 2026-04-23T10:00:00.000Z
google.com, pub-1234567890123456, DIRECT, f08c47fec0942fa0
rubiconproject.com, 25698, RESELLER, 0bfd66d529a55807
advlink.it, adv-0001, DIRECT
```

Conforme a IAB Tech Lab `ads.txt` specification v1.0.2.
